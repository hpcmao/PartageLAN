# État — portage Windows & SSH multi-boot (13/07/2026)

Session de création du **client Windows** (`winjeux`, 10.0.0.5) et de mise en place du **SSH
multi-boot** depuis le Mac `vemao` (10.0.0.4).

## Fait ✅

### Client Windows PartageLAN (`windows/`)
- `partagelan.py` : protocole complet (ping/pong, clip, ls/lsr, get, push/file) + CLI
  (`ping/scan/ls/pull/push/send-clip/listen`). Presse-papier via **API Win32 native (ctypes)**.
  `dir_for_ls` : repli sur l'accueil si le Mac demande un chemin macOS hérité.
- `partagelan_tray.py` : app de fond (pystray + Pillow + tkinterdnd2 + tkinter).
  - Icône zone de notification ; **clic gauche = fenêtre + menu** (surcharge `_on_notify`).
  - **Fenêtre à deux panneaux** (réplique de l'app Mac) : presse-papier bidirectionnel,
    9 thèmes, IP/Tester/Scanner, →/←, dossier de réception, ligne SSH, journal horodaté.
  - **Glisser-déposer** (Explorateur → panneau), **menu clic-droit**, **sélection à la souris**.
  - **Terminal partagé** : `ssh -t <hôte> "export PATH=…/opt/local/bin…; tmux new-session -A -s partagelan"`.
  - Écoute permanente, notifications, **anti-doublon** (mutex), **lancement au démarrage** (registre).
- `PartageLAN.bat` : lanceur silencieux. Testé OK dans les deux sens avec vemao.

### Serveur SSH sur winjeux (Mac → Windows)
- **OpenSSH Server** installé, `sshd` démarrage auto, **shell = PowerShell**.
- Pare-feu : port 22 en entrée, **Profile=Any, RemoteAddress=LocalSubnet** (corrigé : la règle
  par défaut n'était active qu'en profil *Privé*, réseau *Public* → timeout).
- Clé publique **ed25519 de vemao** autorisée dans `C:\ProgramData\ssh\administrators_authorized_keys`
  (winjeux est **admin**, ACL stricte). Compte sans mot de passe → **clé obligatoire**.

## À faire ⏳ (prochaine session)

1. **Valider `ssh winjeux`** depuis vemao avec l'entrée `~/.ssh/config` (`HostKeyAlias winjeux`).
   - Purger une fois l'ancienne empreinte : `ssh-keygen -R 10.0.0.5` (vieille clé de hpcmultimedia).
2. **hpcmultimedia (macOS)** — booter le PC sous macOS, puis :
   - Réglages Système → Général → Partage → **Session à distance**.
   - Depuis vemao : `ssh-copy-id hpcmultimedia`.
3. **pclinux (Linux)** — booter sous Linux, puis :
   - `sudo apt install -y openssh-server && sudo systemctl enable --now ssh` (adapter à la distro).
   - Depuis vemao : `ssh-copy-id pclinux`.
4. Renseigner les **comptes macOS et Linux** dans `~/.ssh/config` (voir `README.md`).

  `ssh` simple pour un shell Windows propre.
