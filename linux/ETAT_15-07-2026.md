# État — portage Linux (15/07/2026)

Session de création du **client Linux** (`haikubuntu`, 10.0.0.5, Ubuntu MATE 24.04, X11),
sur la base du portage Windows (`windows/`). Le nom réseau retenu pour l'OS Linux du PC
multiboot est **haikubuntu** (et non `pclinux` évoqué dans `windows/ETAT_13-07-2026.md`).

## Fait ✅

### Client Linux PartageLAN (`linux/`)
- `partagelan.py` : protocole complet (ping/pong, clip, ls/lsr, get, push/file) + CLI
  (`ping/scan/ls/pull/push/send-clip/listen`), stdlib seule. Presse-papier via
  **xclip** (ou wl-clipboard/xsel/GTK selon disponibilité). `dir_for_ls` : repli sur
  l'accueil pour un chemin macOS (`/Users/...`) ou Windows (`C:\...`) hérité du multiboot.
  Fidélité Mac : une trame `file` entrante **honore `meta.path`** si le dossier existe
  (le portage Windows l'ignorait).
- `partagelan_tray.py` : app de fond (pystray/AppIndicator + Pillow + tkinterdnd2 + tkinter).
  - Icône zone de notification (bleue = Mac en ligne, grise = absent), notifications
    via `notify-send`, anti-doublon par **socket Unix abstraite**.
  - **Fenêtre à deux panneaux** identique aux versions Mac/Windows : presse-papier
    bidirectionnel anti-écho (boucle 0,6 s dans le thread Tk), 9 thèmes (« Système »
    suit gsettings), IP/Tester/Scanner, →/←, dossier de réception, ligne SSH
    (Terminal SSH + Terminal partagé tmux), journal horodaté, glisser-déposer,
    clic-droit, sélection à la souris.
  - **Lancement au démarrage** : `~/.config/autostart/PartageLAN.desktop` (case du menu).
  - Terminal : mate-terminal (puis gnome-terminal/konsole/… en repli) via un petit `.sh`.
  - Réglages : `~/.config/PartageLAN/config.json`.
- `install.sh` : apt (python3-tk, xclip) + venv `--system-site-packages` (pystray,
  pillow, tkinterdnd2) + lanceurs .desktop (menu + autostart) + règle ufw + lancement.
- `PartageLAN.sh` : lanceur double-clic (installe au 1ᵉʳ lancement).

### Testé ✅
- CLI contre le **vrai Mac vemao** : ping (« vemao — macOS 15.7 »), `ls ~`.
- Boucle locale : ping, ls, repli chemin étranger, **push/pull 1 Mo avec sha256
  identiques**, send-clip (le presse-papier survit à la mort du processus grâce au
  gestionnaire de presse-papier de MATE).
- App de fond : fenêtre vérifiée à l'écran (panneau distant branché sur vemao),
  écoute 7365 OK, anti-doublon OK, install.sh idempotent.

### Réglages à la demande (même session) ✅
- **Clic gauche** sur l'icône = ouvrir la fenêtre **et** afficher le menu (surcharge
  `TrayIcon._on_status_icon_activate`, fidèle à la version Windows) ; clic droit = menu.
- **Affichage ×1,5** : polices nommées Tk à 15 pt (tout texte), en-têtes 17 pt,
  journal Monospace 14, listes rowheight 33 + icônes 24 px, colonnes élargies.
- **Réception = dossier affiché à gauche** (le réglage « dossier de réception » fixe a
  été supprimé) ; le panneau se rafraîchit à la réception ; `meta.path` explicite reste
  honoré.
- **Persistance immédiate** : chemins des deux panneaux à chaque navigation, taille et
  position de fenêtre à chaque déplacement/redimensionnement (débounce 0,5 s) —
  robuste au crash/kill.
- **Terminal SSH / partagé** : correction d'un bug de quoting (le `.sh` local aurait
  expansé `$PATH`/`$SHELL` côté Linux ; désormais `shlex.quote` → expansion sur la
  machine distante). `tmux` 3.4 installé sur haikubuntu (sens Mac → haikubuntu).
- **Tri par date de modification** (plus récent en premier) + colonne « Modifié » :
  champ `mtime` ajouté aux entrées `lsr` (l'app Mac ignore les champs inconnus ;
  elle n'envoie pas de mtime → son listing garde l'ordre Mac). CLI `ls` affiche
  aussi les dates.

### Réseau / système (via SudoManager, profils Install + Admin) ✅
- **Pare-feu** : `ufw allow 7365/tcp comment 'PartageLAN'` (IPv4+v6) — le Mac peut
  joindre haikubuntu.
- **SSH entrant** : `openssh-server` installé, service `ssh` actif au démarrage,
  port 22 ouvert **LAN seulement** (`ufw allow from 10.0.0.0/24 to any port 22`).
  Empreinte hôte ed25519 : `SHA256:DVRivsZjdlP/fu5g5RhLv+9ByuFB6h2/cvFLc28NNK4`.

### SSH bidirectionnel par clé (fin de session) ✅
- haikubuntu → vemao : clé `id_ed25519` de haikubuntu autorisée sur le Mac
  (Terminal SSH / partagé sans mot de passe). Port 7365 de haikubuntu **vérifié
  joignable depuis le Mac** (règle ufw OK de bout en bout) ; tmux 3.7b sur le Mac.
- vemao → haikubuntu : clé du Mac dans `authorized_keys` ; **trois alias multiboot**
  (`haikubuntu`/`winjeux`/`hpcmultimedia`, avec `HostKeyAlias`) ajoutés au
  `~/.ssh/config` du Mac (sauvegarde `config.bak.*`), entrée périmée de
  `known_hosts` purgée, clé d'hôte de haikubuntu pré-enregistrée.
  Depuis le Mac : `ssh haikubuntu` direct (vérifié en BatchMode).

## Notes / pièges
- ufw est **actif** sur haikubuntu (contrairement au pare-feu Windows où seule la règle
  manquait) ; sans la règle, `bind` fonctionne mais rien n'entre depuis le LAN.
- **Icône tray** : piège pystray — `run_detached()` ne démarre AUCUNE boucle GLib
  (il suppose que l'appli en fait tourner une ; or la nôtre est en Tk) → les
  `GObject.idle_add` ne s'exécutent jamais et l'icône n'apparaît pas, quel que soit
  le backend. Remède : `icon.run()` dans un **thread dédié**. On force aussi
  `PYSTRAY_BACKEND=gtk` sous X11 (zone de notification XEmbed, plus fiable que
  AppIndicator sous MATE) → **clic gauche = fenêtre + menu**, clic droit = menu,
  comme sur Windows.
- Relancer l'app (lanceur/PartageLAN.sh) quand elle tourne déjà **ouvre sa fenêtre**
  (demande passée par la socket anti-doublon).
- La zone de notification MATE **empilait 2 petites icônes par colonne** (la nôtre
  sous celle de gestion des écrans). Corrigé via
  `dconf write /org/mate/panel/objects/notification-area/prefs/min-icon-size 32`
  (annulable par `dconf reset` de la même clé).
- SudoManager : profils « Install » (apt) et « Admin » (ufw/systemctl).
