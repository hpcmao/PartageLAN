# Etat du Projet - PartageLAN
**Date:** 12-07-2026 23h45

App macOS (SwiftUI) de partage entre deux Macs du LAN (10.0.0.4 `vemao` ↔ 10.0.0.5
`hpcmultimedia`) : deux panneaux de navigation façon Transmit, copie de fichiers dans les
deux sens, presse-papier partagé. TCP sur le port **7365**. App **agent** (barre de menus).

## Fonctionnalites implementees
- [x] Deux panneaux (local / distant) avec navigation complète (double-clic, parent, chemin éditable, refresh)
- [x] Copie de fichiers bidirectionnelle (→ / ←, double-clic récupérer, drag & drop, dédoublonnage)
- [x] Sélection multiple dans les panneaux : clic simple (remplace), ⌘-clic (bascule), ⇧-clic
      (plage depuis la dernière ancre), **glisser la souris pour sélectionner un rectangle de
      lignes façon Finder** (⌘-glisser = additif). Remplace la sélection native de `List`, qui
      ne supporte pas le glisser sur macOS — geste maison (`rowGesture(for:)`) avec rectangle
      visuel pendant le glisser. Affecte les deux panneaux (composant `PaneView` partagé).
- [x] Presse-papier partagé texte (Les 2 sens / Envoi / Réception / Coupé, sondage 0,6 s, anti-écho)
- [x] Dossier de réception configurable (défaut ~/Downloads)
- [x] Thèmes : Système, Clair, Sombre, Océan, Sépia, **Nord, Dracula, Solarisé clair, Contraste
      élevé** (ajoutés le 12-07-2026 — palettes connues + un thème pensé daltonisme/fort contraste,
      accent orange/ambre sur fond quasi noir pour rester distinguable quel que soit le type de
      daltonisme).
- [x] Scan réseau : bouton « Scanner » (panneau distant) → sonde le /24, menu déroulant des Macs détectés, sélection auto si un seul
- [x] Icône barre de menus macOS (MenuBarExtra) : statut du pair + actions rapides
- [x] Mode agent (LSUIElement) : pas d'icône Dock, fenêtre non ouverte au démarrage (defaultLaunchBehavior .suppressed), ouverte à la demande
- [x] Bouton Terminal SSH : ouvre Terminal.app en ssh -t vers user@hôte + dossier distant (champs mémorisés), rejoint une session tmux partagée (`partagelan`) plutôt qu'un shell simple
- [x] Panneau distant, par ligne : **clic droit (menu contextuel) ET bouton ⋯▾ dédié (menu
      déroulant au clic gauche)** → Terminal SSH à cet endroit / Ouvrir dans le Finder (smb://).
      Les deux chemins déclenchent les mêmes actions (`onOpenSSH`/`onOpenFinder`), ajout du
      12-07-2026 : le clic droit existait déjà (10-07-2026), le bouton ⋯▾ est nouveau.
- [x] Bouton Terminal partagé : rejoint/crée en local (sans SSH) la même session tmux `partagelan` — permet de voir/piloter en direct un Terminal SSH ouvert vers cette machine depuis l'autre Mac, dans les deux sens. Popup explicatif (OK/Annuler) avant ouverture. Nécessite tmux installé (`sudo port install tmux` sur macOS avec MacPorts) — pas de repli si absent, message d'erreur dédié dans le terminal (code 127)
- [x] Persistance : chemins panneaux, dossier réception, IP pair, thème, géométrie, sshHost/sshDir (didSet UserDefaults)
- [x] Installation/màj en un clic : `PartageLAN.command` (setup_signing + build + copie /Applications + LaunchAgent)
- [x] Lancement automatique à la session (LaunchAgent `fr.vemao.partagelan`)
- [x] Signature de code stable (certif auto-signé `PartageLAN Self-Signed`) → Little Snitch/Gatekeeper gardent leurs règles entre rebuilds

## Architecture (rappel)
- Transport : `NWListener`/`NWConnection` TCP, port 7365 (choix arbitraire, cst `portNumber`). Trame = `[UInt32 BE longueur][JSON Meta][octets]`.
- Types de messages : `clip`, `file`, `ping`/`pong`, `ls`/`lsr`, `get`, `err`.
- `PartageEngine` (ObservableObject) : moteur réseau + état. `PaneModel` : un panneau.
- `PaneView` : ligne rendue par `row(for:)`, sélection gérée entièrement par `rowGesture(for:)`
  (un seul `DragGesture(minimumDistance: 0)` par ligne — remplace clic natif de `List` ET
  l'ancien `onTapGesture(count: 2)` pour éviter tout conflit d'arbitrage de gestes SwiftUI ;
  double-clic détecté par horodatage maison, pas par un second geste concurrent). Rectangle de
  sélection calculé via `RowFrameKey` (PreferenceKey collectant le frame de chaque ligne dans
  l'espace de coordonnées nommé `"paneList"`). Bouton ⋯▾ (menu déroulant SSH/Finder) posé en
  frère du contenu cliquable, pas en enfant, pour ne pas hériter du geste de sélection.
- Scan : `scanNetwork()` balaie .1–.254 (concurrence 32 via semaphore), `pingHost()` par hôte (timeout 1 s).
- Presse-papier : `pollClipboard()` (timer 0,6 s), `applyRemoteClipboard()`. Anti-boucle via `lastRemoteText` + `lastChangeCount`.
- Menu bar : `MenuBarExtra` / `MenuBarContent`. Fenêtre : `Window(id:"main")` + `defaultLaunchBehavior(.suppressed)`, ouverte via `openWindow`.
- SSH/Terminal : jamais d'Apple Events/NSAppleScript (autorisation « Automatisation » peu fiable pour une app agent LSUIElement). À la place : script `.command` temporaire écrit sur disque + `open -a Terminal script.command`. `openSSHTerminal(host:dir:)` lance `ssh -t host 'tmux new -A -s partagelan [-c dir]'` ; `openSharedTerminal()` fait la même chose en local sans SSH. Même session tmux nommée des deux côtés → partage bidirectionnel.
- Finder distant : `openRemoteFinder(path:)` ouvre une URL `smb://user@ip/partage` via `NSWorkspace.shared.open` (pas d'Apple Events non plus).

## Signature / Little Snitch
- Certif `PartageLAN Self-Signed` créé par `setup_signing.sh` (openssl EKU codeSigning → p12 → security import -A).
- OpenSSL 3 exige `-legacy` pour le p12 ; LibreSSL non (le script détecte via `openssl version`).
- Le certif doit être créé **localement** sur chaque Mac (trousseau inaccessible en SSH — `codesign` échoue aussi en SSH même certif déjà présent, pas seulement à la création).
- Après passage ad-hoc→stable, autoriser PartageLAN **une dernière fois** dans Little Snitch ; ensuite plus de re-blocage.

## Dépendances externes (hors app)
- **tmux**, requis pour le Terminal partagé. Pas bundlé, pas installé par défaut sur macOS.
  hpcmultimedia utilise **MacPorts** (`sudo port install tmux`), pas Homebrew (absent sur cette
  machine). Statut sur vemao à revérifier si besoin (MacPorts confirmé présent aussi, tmux
  installé le 10-07-2026).
